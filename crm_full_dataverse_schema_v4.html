Design decisions

Staff vs system user

Staff is a custom table with a SystemUserID lookup back to Dataverse's built-in User table. This lets you map the logged-in user (User().Email) to their Staff record at runtime — enabling auto-populated CreatedBy, personal dashboards, and caseload filtering without hardcoding.

StaffStudent junction

StaffStudent has both an AssignmentType choice and an IsPrimary boolean. The boolean enables a fast single-row lookup for a student's primary counselor without parsing choice values. A student can have multiple staff assigned with different roles simultaneously.

CourseOfferings — pivotal junction

Courses holds the master record (ENGL 123). CourseOfferings is the actual instance — Section 02, MWF 10am, Fall 2024. Student enrollments, instructor assignments, and accommodation requests all point at an offering, not a course directly.

Eligibility vs. requests

StudentEligibility = what a student has been assessed and cleared to receive (stable, set by staff). AccommodationRequests = active registrations actually invoked per offering or term. Comparing the two gives "eligible but not yet registered" — a key workflow view.

AccommodationRequests.OfferingID

The OfferingID FK is optional by design. Course-specific accommodations (extended exam time for ENGL 123) link to an offering. General accommodations (parking, housing) do not. One table handles both without a split.

StudentNotes — appointment link

StudentNotes has an optional AppointmentID FK. A note can be a standalone general observation on a student, or tied to a specific appointment for context. The IsPrivate boolean restricts visibility to the authoring staff member only — useful for sensitive observations that shouldn't be visible to all staff.

Documents — student only

Documents attaches directly to a student record. Without case management, all uploaded files (diagnostic letters, medical docs, correspondence) are stored at the student level and browsable from the student detail screen.

Data dictionary
StudentsCore student profile. One record per student registered with DAS. Acts as the primary hub — every other table has a direct or indirect relationship back here.
StudentIDPK · AutonumberSystem-generated unique identifier. Used as the FK target in all related tables.
FirstName / LastNameTextStudent's legal name. Combined as a calculated Full Name column for display throughout the app.
EmailTextUniversity-issued email address. Primary contact method and login identifier lookup.
PhoneTextPrimary phone number. Stored as text to accommodate formatting variations.
UniversityIDTextInstitutional student ID. Used as the matching key when syncing enrollment data from PeopleSoft via Power Automate.
ProgramTextStudent's declared academic program or major. Free text to avoid maintaining a program lookup table.
EnrollmentStatusChoiceActive = currently enrolled. Graduated = degree awarded. Withdrawn = left voluntarily. Leave of Absence = temporarily away.
StaffDAS office staff who interact with students and manage accommodations. Linked to Dataverse's built-in User table via SystemUserID to map the logged-in Microsoft account to the correct staff record at runtime.
StaffIDPK · AutonumberReferenced by StudentStaff, StudentNotes, Appointments, StudentEligibility, and AccommodationRequests.
FirstName / LastName / Email / PhoneTextStaff contact info. Email must match the Microsoft 365 account for SystemUserID mapping to work.
Title / DepartmentTextJob title and department. Used for display on student profiles and in reporting filters.
RoleChoiceFunctional role within DAS. Drives access control — Directors see all records, Counselors see their assigned caseload only.
IsActiveBooleanInactive staff are excluded from assignment dropdowns but records are preserved for historical accuracy.
SystemUserIDLookup → UserLookup to Dataverse's built-in User table. Enables User().Email lookups at runtime to auto-resolve the logged-in user to their Staff record.
StudentStaffJunction table associating students with one or more DAS staff members. Supports multiple concurrent assignments with distinct roles. IsPrimary enables fast lookup of the primary counselor without filtering on choice values.
StudentIDFK → StudentsThe student being assigned.
StaffIDFK → StaffThe staff member assigned to the student.
AssignmentTypeChoicePrimary Counselor = main contact. Secondary = supporting role. Intake Coordinator = managing initial registration. Case Reviewer = oversight only.
AssignedDate / EndDateDateDate range for the assignment. EndDate is null for active assignments.
IsPrimaryBooleanTrue for the student's primary counselor. Only one row per student should have IsPrimary = true at any time. Enforced at the app layer.
SemestersReference table of academic terms. Created once per semester by an administrator. IsActive flags the current term used to pre-populate semester dropdowns throughout the app.
SemesterIDPK · AutonumberReferenced by StudentSemesters and CourseOfferings.
NameTextDisplay name (e.g. "Fall 2024"). Used in dropdowns and report labels.
TermChoiceSeason: Fall, Spring, Summer, Winter. Combined with Year to uniquely identify a semester.
YearIntegerFour-digit academic year (e.g. 2024).
StartDate / EndDateDateOfficial semester boundaries. Used to determine whether accommodations and enrollments fall within an active period.
IsActiveBooleanTrue for the current semester. Only one semester should be active at a time. Drives default selection in course and enrollment forms.
StudentSemestersRecords which semesters a student is registered with DAS. Distinct from university enrollment — a student can be enrolled at the university but not actively registered with DAS in a given term.
StudentIDFK → StudentsThe student registered in the semester.
SemesterIDFK → SemestersThe semester the student is registered in.
RegistrationDateDateDate the student registered with DAS for this term. Distinct from the semester's StartDate.
StatusChoiceActive = currently registered. Completed = term ended with services used. Withdrawn = left DAS mid-term.
ProfessorsReference table of university instructors. Populated via the PeopleSoft sync flow. DAS staff may contact professors regarding student accommodations — office location and phone support this workflow.
ProfessorIDPK · AutonumberReferenced by CourseInstructors.
FirstName / LastName / Email / PhoneTextContact information. Email is the primary channel for notifying instructors of approved accommodations.
DepartmentTextAcademic department. Used for filtering in instructor lookup screens.
CoursesMaster catalog of courses. Each record represents a course in the abstract (ENGL 101 always exists) independent of when it is offered. Populated via PeopleSoft sync. DeptTitle and CourseCode are stored separately to enable department-level filtering.
DeptKeyComposite KeyDepartment abbreviation (e.g. "ENGL", "MATH"). Stored separately from CourseCode to support department-level filtering.
CourseCodeComposit KeyNumeric or alphanumeric course number (e.g. "101"). Combined with DeptTitle in the app: DeptTitle & " " & CourseCode.
CourseTitleTextOfficial course title (e.g. "Introduction to Composition"). Displayed alongside course code in student course lists.
CreditsIntegerCredit hours awarded. Informational — not used in accommodation logic.
DeliveryFormatChoiceIn-Person, Online, or Hybrid. Relevant to accommodation planning — some accommodations apply only to in-person or hybrid courses.
CourseOfferingsA specific instance of a course in a given semester. The pivotal junction in the academic cluster — student enrollments, instructor assignments, and accommodation requests all link here rather than to Courses directly. Enables querying all accommodations active for a student in a specific semester.
OfferingIDPK · AutonumberReferenced by CourseInstructors, StudentCourses, and AccommodationRequests.
DeptKeyFK → CoursesPart of composite key for inique course.
CourseCodeFK → CoursesPart of composite key for inique course.
SemesterIDFK → SemestersThe semester in which this offering runs.
SectionTextSection identifier (e.g. "001"). Together with CourseID and SemesterID, uniquely identifies an offering.
RoomTextBuilding and room where the course meets. Used when coordinating testing accommodations with physical location.
MeetingDays / MeetingTimeTextSchedule information (e.g. "MWF", "09:00–09:50"). Displayed on student course lists and accommodation views.
CourseInstructorsJunction linking professors to course offerings with a role designation. Supports multiple instructors per offering. Role is stored at the junction level because the same person can hold different roles across different offerings.
OfferingIDFK → CourseOfferingsThe offering being taught.
ProfessorIDFK → ProfessorsThe instructor assigned to the offering.
RoleChoicePrimary Instructor = lead instructor of record. Co-Instructor = shared teaching. Teaching Assistant = supporting role, typically not the accommodation contact.
StudentCoursesRecords a student's enrollment in a specific course offering. Populated via the PeopleSoft sync flow. Dropped and withdrawn rows are retained rather than deleted to maintain a full enrollment history.
StudentIDFK → StudentsThe enrolled student.
OfferingIDFK → CourseOfferingsThe specific section and semester the student is enrolled in.
EnrollmentStatusChoiceEnrolled = active. Dropped = withdrew during add/drop. Withdrawn = left after deadline. Completed = finished. Audit = attending without credit.
EnrolledDateDateDate the student registered for the course. Synced from PeopleSoft.
DropDateDateDate the student dropped course if applicable. Synced from PeopleSoft.
CurentGradeStatusChoicePassed, Withdrew, Failed, Incomplete status. Synced from PeopleSoft.
GradeApprovalboolUsed to flag if grade needs to be disputed for accessibility compliance.
AccommodationTypesReference table defining the master list of available accommodation types. Maintained by administrators — not editable by general staff from the Canvas App. Both StudentEligibility and AccommodationRequests reference this table to ensure consistent vocabulary across both workflows.
TypeIDPK · AutonumberReferenced by StudentEligibility and AccommodationRequests.
NameTextAccommodation name as displayed in the app and on letters (e.g. "Extended Time — 1.5x").
DescriptionTextPlain-language description of what the accommodation entails. Shown in staff-facing detail views.
RequiresMedicalDocBooleanTrue if supporting medical documentation is required before eligibility can be granted. Surfaced as a warning in the eligibility form.
StudentEligibilityRecords what accommodations a student has been cleared to receive — set by DAS staff after reviewing documentation. Relatively stable and not term-specific. Comparing this against AccommodationRequests for the same student gives the "eligible but not yet registered" view, a key counselor workflow each semester.
StudentIDFK → StudentsThe student granted eligibility.
AccommodationTypeIDFK → AccommodationTypesThe specific accommodation type the student is eligible for.
GrantedDateDateDate DAS formally approved eligibility for this accommodation.
ExpiryDateDateDate eligibility expires, if applicable. Null for indefinite eligibility. Used by Power Automate to flag approaching expirations.
GrantedByFK → StaffStaff member who approved the eligibility. Provides an audit trail of approval decisions.
NotesTextConditions, caveats, or context related to this specific eligibility grant.
AccommodationRequestsActive registrations a student has invoked, per offering or per term. Multiple rows per student are expected. OfferingID is optional — course-specific accommodations link to an offering; general accommodations (parking, housing) do not. One table handles both cases without a split.
StudentIDFK → StudentsThe student making the request.
AccommodationTypeIDFK → AccommodationTypesThe type of accommodation being requested.
OfferingIDFK → CourseOfferings (optional)Links to a specific course section. Null for non-course accommodations such as housing or parking.
StatusChoicePending = awaiting review. Approved = active. Denied = not approved. Expired = past ActiveUntil. Withdrawn = student cancelled.
RegisteredDateDateDate the student submitted the request.
ApprovedByFK → StaffStaff member who approved the request. Null until approval is granted.
SpecialInstructionsTextAccommodation-specific notes for the instructor or testing center (e.g. "Student requires large-print exam materials").
StudentNotesGeneral-purpose notes recording staff observations, contact events, and meeting summaries. A note can stand alone or link to a specific appointment via the optional AppointmentID FK. IsPrivate restricts visibility to the authoring staff member — useful for sensitive observations not appropriate for shared staff views.
StudentIDFK → StudentsThe student this note is about.
StaffIDFK → StaffThe staff member who authored the note. Auto-populated at runtime using the logged-in user's Staff record.
AppointmentIDFK → Appointments (optional)Links the note to a specific appointment for context. Null for standalone notes not tied to a scheduled interaction.
NoteTextMultiline textThe body of the note. No length restriction in Dataverse.
NoteTypeChoiceCategorises the interaction: General, Phone Call, Email, Meeting, Follow-up, Observation. Used to filter the note timeline on the student detail screen.
CreatedOnDateTimeTimestamp of note creation. Auto-populated by the system.
IsPrivateBooleanWhen true, visible only to the authoring staff member. Enforced at the Canvas App filter layer using the logged-in user's StaffID.
DocumentsStores metadata for files attached to a student record. The actual file binary is stored in Dataverse's native file column. All documents attach at the student level — without case management, files are browsable from the student detail screen by type.
StudentIDFK → StudentsThe student this document belongs to.
DocumentTypeChoiceClassifies the file: Diagnostic Letter, Medical Documentation, Accommodation Letter, Correspondence, Other. Used for filtering in the Documents tab.
FileNameTextOriginal filename including extension. Displayed in the documents gallery.
UploadedOnDateTimeTimestamp of upload. Used to sort the documents list newest-first.
AppointmentsScheduled interactions between students and DAS staff. Supports intake sessions, follow-ups, documentation reviews, and general meetings. Notes created during or after an appointment link back here via StudentNotes.AppointmentID, creating a full interaction history per student.
StudentIDFK → StudentsThe student attending the appointment.
StaffIDFK → StaffThe DAS staff member hosting the appointment.
DateTimeDateTimeScheduled date and time. Used by the Power Automate reminder flow which triggers 24 hours before this value.
DurationMinsIntegerPlanned duration in minutes. Displayed on the appointment card.
AppointmentTypeChoiceIntake = first meeting. Follow-up = check-in. Review = formal accommodation review. Documentation = collecting or reviewing paperwork. General = other.
StatusChoiceScheduled = upcoming. Completed = held. Cancelled = cancelled by either party. No Show = student did not attend. Status changes trigger the notification flow.
